Zadania edukacyjne – Gra „Czerwona Kulka” (Python + Pygame)

Ten zestaw zadań pomaga krok po kroku rozwijać prostą grę w Pythonie (Pygame), w której sterujemy czerwoną kulką. Każde zadanie można robić osobno z dzieckiem: czytając opis, a potem wprowadzając zmiany w kodzie.

Zadanie 1 – Dodanie dźwięku skoku

Cel

Gdy kulka skacze, odtwarzany jest dźwięk. Uczeń uczy się: jak ładować dźwięk, jak go odtwarzać przy konkretnym zdarzeniu.

Krok 1 – Przygotuj plik dźwiękowy

  1. Znajdź krótki dźwięk w formacie WAV (np. efekt „click”, „jump”).
  2. Zapisz go w tym samym folderze, w którym znajduje się plik z grą.
  3. Nazwij plik: skok.wav.

Krok 2 – Włącz moduł dźwięku w Pygame

W funkcji main(), zaraz po linii:

pygame.init()

dodaj linię:

pygame.mixer.init() # włącza obsługę dźwięku

Krok 3 – Wczytaj dźwięk skoku

W funkcji main(), po utworzeniu okna i zegara, znajdź fragment:

ekran = pygame.display.set_mode((SZEROKOSC, WYSOKOSC)) pygame.display.set_caption("Czerwona Kulka - Dino Style") zegar = pygame.time.Clock()

Pod nim dopisz:

# ŁADOWANIE DŹWIĘKU SKOKU dzwiek_skok = pygame.mixer.Sound("skok.wav")

Krok 4 – Zagraj dźwięk przy skoku

Znajdź w pętli zdarzeń taki fragment:

if event.type == pygame.KEYDOWN: if event.key in (pygame.K_SPACE, pygame.K_UP): if not w_powietrzu: predkosc_y = -18 w_powietrzu = True

Zamień go na:

if event.type == pygame.KEYDOWN: if event.key in (pygame.K_SPACE, pygame.K_UP): if not w_powietrzu: predkosc_y = -18 w_powietrzu = True dzwiek_skok.play() # zagraj dźwięk skoku

Po tym zadaniu warto porozmawiać z uczniem: „Co się stanie, jeśli zmienimy plik WAV?”, „Czy możemy mieć różne dźwięki dla różnych akcji?”.

Zadanie 2 – Zmiana kolorów tła i podłogi

Cel

Uczeń uczy się, że kolory w komputerze to trójki liczb (RGB), i że łatwo można odmienić wygląd gry.

Krok 1 – Dodaj nowe kolory

Na początku pliku, obok definicji kolorów:

BIALY = (255, 255, 255) CZARNY = (0, 0, 0) CZERWONY = (200, 0, 0)

dodaj np.:

NIEBIESKIE_TLO = (150, 200, 255) ZIELONA_PODLOGA = (0, 180, 0)

Krok 2 – Zmień kolor tła

Znajdź linijkę:

ekran.fill(BIALY)

Zamień ją na:

ekran.fill(NIEBIESKIE_TLO)

Krok 3 – Zmień kolor podłogi

Znajdź linijkę, która rysuje podłogę:

pygame.draw.line(ekran, CZARNY, (0, podloga_y), (SZEROKOSC, podloga_y), 2)

Zamień ją na:

pygame.draw.line(ekran, ZIELONA_PODLOGA, (0, podloga_y), (SZEROKOSC, podloga_y), 4)

Zachęta: poproś ucznia, żeby sam wymyślił swoje kolory tła i podłogi (inne wartości RGB) i zapisał je np. jako „PIASKOWE_TLO”, „FIOLETOWA_PODLOGA” itd.

Zadanie 3 – Mrugające oko

Cel

Kulka co jakiś czas mruga okiem. Uczeń uczy się pracy z czasem, stanami (prawda/fałsz) oraz prostą animacją.

Krok 1 – Dodaj zmienne do mrugania

W funkcji main(), obok:

licznik_klatek = 0 faza = 0

dodaj:

mruga = False czas_nastepnego_mrugu = pygame.time.get_ticks() + 2000 # pierwszy mrug za ok. 2 sekundy

Krok 2 – Aktualizuj stan mrugania

W głównej pętli gry (po wynik += 1) dopisz:

# MRUGANIE OKIEM teraz_czas = pygame.time.get_ticks() if teraz_czas > czas_nastepnego_mrugu: # przełączamy stan mrugania mruga = not mruga if mruga: # oko zamknięte przez chwilę czas_nastepnego_mrugu = teraz_czas + 150 # 0.15 sekundy else: # kolejne mrugnięcie za 2–4 sekundy czas_nastepnego_mrugu = teraz_czas + random.randint(2000, 4000)

Krok 3 – Przekaż informację o mruganiu do funkcji rysującej

Zmień nagłówek funkcji rysuj_kulke z:

def rysuj_kulke(ekran, x, y, faza):

na:

def rysuj_kulke(ekran, x, y, faza, mruga):

Następnie znajdź wywołanie tej funkcji:

rysuj_kulke(ekran, kulka_x, kulka_y, faza)

i zamień na:

rysuj_kulke(ekran, kulka_x, kulka_y, faza, mruga)

Krok 4 – Zmień rysowanie oka

W funkcji rysuj_kulke znajdź fragment rysujący oko:

pygame.draw.circle(ekran, CZARNY, (oko_x, oko_y), 4)

Zastąp go tym kodem:

if mruga: # oko zamknięte - kreska pygame.draw.line(ekran, CZARNY, (oko_x - 4, oko_y), (oko_x + 4, oko_y), 2) else: # normalne oko - kółko pygame.draw.circle(ekran, CZARNY, (oko_x, oko_y), 4)

Spróbuj z uczniem zmienić czas mrugania (150 ms, 500 ms itp.) oraz częstotliwość (np. co 1 sekundę).

Zadanie 4 – System żyć

Cel

Po zderzeniu z przeszkodą kulka nie ginie od razu – ma np. 2 życia. Uczeń poznaje licznik żyć, warunki i reset pozycji.

Krok 1 – Dodaj licznik żyć

W funkcji main(), obok wynik = 0, dopisz:

zycia = 2 # liczba żyć gracza

Krok 2 – Zmień zachowanie przy kolizji

Znajdź fragment z kolizją:

for przes in przeszkody: if kulka_rect.colliderect(przes): uruchomiona = False

Zastąp go poniższym kodem:

for przes in przeszkody: if kulka_rect.colliderect(przes): zycia -= 1 # tracimy jedno życie przeszkody.clear() # usuwamy przeszkody z ekranu kulka_x = 100 kulka_y = podloga_y - 30 predkosc_y = 0 w_powietrzu = False if zycia <= 0: uruchomiona = False # koniec gry, jeśli brak żyć break # ważne: przerywamy pętlę, żeby nie odjąć kilku żyć naraz

Krok 3 – Wyświetl ilość żyć na ekranie

Po wyświetleniu wyniku:

tekst = czcionka.render(f"Wynik: {wynik}", True, CZARNY) ekran.blit(tekst, (10, 10))

dodaj:

tekst_zycia = czcionka.render(f"Życia: {zycia}", True, CZARNY) ekran.blit(tekst_zycia, (10, 40))

Zapytaj ucznia: „A co, jeśli damy 5 żyć?”, „Czy możemy dodać serduszka zamiast liczby?”.

Zadanie 5 – Ekran „Game Over” i restart

Cel

Po utracie wszystkich żyć gra się zatrzymuje i wyświetla ekran „Game Over”. Gracz może nacisnąć SPACJĘ, aby zagrać ponownie.

Krok 1 – Dodaj zmienną stanu gry

W funkcji main(), obok:

wynik = 0 zycia = 2

dodaj:

game_over = False

Krok 2 – Ustaw game_over po utracie żyć

W kodzie, gdzie sprawdzamy:

if zycia <= 0: uruchomiona = False

zamień to na:

if zycia <= 0: game_over = True

Krok 3 – Obsłuż ekran Game Over w pętli gry

Na początku pętli while uruchomiona:, zaraz po zegar.tick(FPS), można dodać blok:

if game_over: # RYSUJEMY EKRAN KOŃCOWY ekran.fill(BIALY) tekst_koniec = czcionka.render("GAME OVER", True, CZARNY) instrukcja = czcionka.render("Naciśnij SPACJĘ, aby zagrać jeszcze raz", True, CZARNY) ekran.blit(tekst_koniec, (SZEROKOSC//2 - 80, WYSOKOSC//2 - 40)) ekran.blit(instrukcja, (SZEROKOSC//2 - 200, WYSOKOSC//2)) pygame.display.flip() # obsługa zdarzeń tylko dla ekranu końcowego for event in pygame.event.get(): if event.type == pygame.QUIT: uruchomiona = False if event.type == pygame.KEYDOWN and event.key == pygame.K_SPACE: # reset gry przeszkody.clear() wynik = 0 zycia = 2 kulka_x = 100 kulka_y = podloga_y - 30 predkosc_y = 0 w_powietrzu = False game_over = False continue # pomijamy resztę logiki, dopóki trwa game_over

To zadanie jest trudniejsze. Warto robić je razem z dzieckiem, tłumacząc, czym jest „stan gry” i dlaczego używamy continue.

Podpowiedź dla rodzica / nauczyciela:
Nie trzeba robić wszystkich zadań naraz. Można wydrukować tę kartę i odhaczać z dzieckiem kolejne zadania, dopisując na marginesach własne pomysły i eksperymenty (np. inne dźwięki, kolory, więcej żyć).
